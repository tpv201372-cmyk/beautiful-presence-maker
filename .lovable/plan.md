

## Мобильная версия раздела «Отзывы учеников»: горизонтальный слайдер

### Что меняем
В `src/components/landing/Testimonials.tsx` добавляем адаптивную подачу:
- **Мобильный (< md)** — горизонтальный слайдер с пролистыванием в бок (как в `Cases.tsx`).
- **Планшет и десктоп (≥ md)** — текущая masonry-сетка без изменений.

### Дизайн мобильного слайдера

```text
┌─────────────────────────────┐
│ ОТЗЫВЫ УЧЕНИКОВ        ‹ › │
│ Что говорят те, кто         │
│ уже внутри                  │
│                             │
│  ┌───────────┐ ┌─────────   │
│  │ скрин 1   │ │ скрин 2    │
│  │ (полный)  │ │ (полный    │
│  │           │ │            │
│  └───────────┘ └─────────   │
│                             │
│   ● ● ○ ○ ○ ○ ○ ○ ○ ○ ○    │
└─────────────────────────────┘
```

- Используем уже подключённый `Carousel` (`embla-carousel-react`) — тот же компонент, что в `Cases.tsx`.
- `basis-[85%]` на мобильном — следующая карточка слегка «выглядывает» справа, подсказывая возможность свайпа.
- Высота карточки естественная (`object-contain`, `h-auto`), без обрезки — отзыв читается целиком.
- Стрелки навигации (‹ ›) в верхнем правом углу секции, точки-индикаторы под слайдером — точно как в Cases.
- Стиль карточки сохраняем: `bg-cream`, `border border-border`, золотая кавычка в углу, hover-эффекты.

### Адаптивность

- На мобильном (`< md`): рендерим `<Carousel>` с горизонтальным скроллом.
- На `md+`: рендерим текущую masonry (`columns-2 lg:columns-3 xl:columns-4`).
- Переключение через CSS-классы `md:hidden` / `hidden md:block` — обе разметки в JSX, но видна только нужная. Это проще и надёжнее, чем использовать `useIsMobile` (избегаем мерцания при гидратации SSR).

### Технические детали

- Файл: только `src/components/landing/Testimonials.tsx`.
- Импорты добавить: `Carousel`, `CarouselContent`, `CarouselItem`, `type CarouselApi` из `@/components/ui/carousel`; `useState`, `useEffect`; `ArrowRight` из `lucide-react`.
- Логика управления API карусели (стрелки + активная точка) — копируем паттерн из `Cases.tsx` (`api.on("select", ...)`).
- Массив `testimonials` и сами импорты картинок не меняются.
- Никаких новых зависимостей.
