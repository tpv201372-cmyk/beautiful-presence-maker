

## Объединить Anna + Алёна + Guzel-course на одном слайде

### Что меняем
В `src/components/landing/Testimonials.tsx`, в массиве `mobileSlides` для мобильного слайдера:

**Сейчас (5 пустых слотов под фото):**
- Слайд 1: `anna` + `alena` (короткий стек, остаётся пустое место снизу)
- Слайд 4: `guzelCourse` (одиночный короткий, тоже пустое место снизу)

**Станет:**
- Слайд 1: `anna` + `alena` + `guzelCourse` (тройной стек — заполняет всю высоту)
- Слайд 4 удаляем

### Финальная раскладка `mobileSlides`
1. `anna` + `alena` + `guzelCourse` (стек из 3-х)
2. `leysan` + `adel` (стек)
3. `aliya` + `elena` (стек)
4. `evdokia` (одиночный, длинный)
5. `chinaTruck` (одиночный, с фото)
6. `firstParcel` (одиночный, с фото)
7. `kate` (одиночный, длинный)

Итого 7 слайдов вместо 8 — счётчик точек обновится автоматически (`count` берётся из `api.scrollSnapList().length`).

### Уменьшение размера для умещения
Чтобы тройной стек (anna + alena + guzel) не был выше «эталонного» длинного слайда (kate), для группы из 3-х карточек применяем сжатие изображений:
- Если `group.length === 3` → каждой `<figure>` добавляем класс с `max-h` (например, `max-h-[28vh]`) и `<img>` получает `object-contain h-full w-auto mx-auto`.
- Для групп из 1–2 карточек поведение не меняется (`h-auto`, естественный размер).

Это даёт чуть меньшие, но полностью читаемые скриншоты Anna/Алёны/Guzel в пределах одного экрана.

### Технические детали
- Один файл: `src/components/landing/Testimonials.tsx`.
- Меняется только массив `mobileSlides` (объединение 1-го и 4-го слайдов) и условный класс высоты для тройных групп внутри `CarouselItem`.
- Десктопная masonry-сетка не трогается.
- Без новых зависимостей и ассетов.
