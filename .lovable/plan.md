
## Кейсы учеников: добавить фото «было/стало» для Азамата + слайдер

### Что меняем в разделе «Кейсы учеников» (`src/components/landing/Cases.tsx`)

**1. Добавить фото скриншотов аккаунтов Азамата**
- `user-uploads://IMAGE_2026-04-21_14_14_55.jpg` → «Было» (304 подписчика, @ohtwentysix)
- `user-uploads://IMAGE_2026-04-21_14_14_52.jpg` → «Стало» (12,2 тыс. подписчиков, @azamat_china)
- Файлы скопируем в `src/assets/cases/azamat-before.jpg` и `azamat-after.jpg`, импортируем как ES6-модули.
- Поле `images?: { before: string; after: string }` добавим в данные кейса. У других кейсов фото пока нет — карточка будет работать и без них.

**2. Превратить сетку в горизонтальный слайдер (карусель)**
- Используем уже установленный `src/components/ui/carousel.tsx` (Embla).
- На мобильных и десктопе кейсы пролистываются вбок. Видно ~1 карточку на мобильном, ~2 на md, ~3 на lg.
- Кнопки «назад/вперёд» в стиле сайта (navy/gold, прямоугольные, без скруглений), расположены справа над каруселью рядом с заголовком.
- Снизу — точки-индикаторы текущего слайда (gold для активной, navy/20 для остальных).
- Сохраняем reveal-анимацию появления секции.

**3. Дизайн карточки Азамата с фото**
Структура карточки (cream-фон, тонкая рамка, hover → navy border — как сейчас):
```text
┌─────────────────────────────────┐
│  TAG (gold, uppercase)          │
│  Имя (font-display, navy)       │
│  ─────────────────              │
│  ┌──────────┐  →  ┌──────────┐  │
│  │ ФОТО     │     │ ФОТО     │  │
│  │ "БЫЛО"   │     │ "СТАЛО"  │  │
│  │ оверлей  │     │ оверлей  │  │
│  └──────────┘     └──────────┘  │
│  Текст «Было» → Текст «Стало»   │
└─────────────────────────────────┘
```
- Два изображения в `aspect-[3/4]`, `object-cover`, тонкая рамка border-navy/15.
- Полупрозрачный градиент-оверлей снизу с лейблом «БЫЛО» / «СТАЛО» (uppercase, tracking-wider, маленький).
- Между фото — золотая стрелка `ArrowRight` (как уже используется).
- Под фото оставляем существующие текстовые блоки «Было / Стало», но компактнее.

Карточки без фото (Ильвина, Диана) остаются в текущем виде — без блока с изображениями, чтобы высота карусели была ровной, добавим небольшой spacer/иконку-плейсхолдер в стиле существующего иероглифа `成功` золотым на cream-фоне (низкая прозрачность) — карточки визуально согласованы.
