

## Мобильный слайдер отзывов: группировка коротких отзывов на один слайд

### Проблема
Сейчас на мобильном каждый отзыв = один слайд с `basis-[85%]`. Когда скриншот короткий (например, leysan), под ним остаётся большое пустое пространство до следующей страницы, потому что высота слайда тянется по самому высокому отзыву в карусели (Kate — самый длинный).

### Решение
Объединять подряд идущие **короткие** отзывы в один вертикальный стек на одном слайде. Длинные отзывы (Kate, china-truck с фото, evdokia) остаются по одному на слайде — их не трогаем.

### Группировка слайдов (мобильная карусель)

Классифицируем отзывы по визуальной высоте (по факту скриншотов):
- **Длинные / с фото** (1 на слайд): `kate`, `china-truck`, `first-parcel`, `evdokia`
- **Короткие** (можно по 2 на слайд): `anna`, `alena`, `leysan`, `adel`, `aliya`, `elena`, `guzel-course`

Финальная раскладка слайдов:
1. `anna` + `alena` (стек)
2. `leysan` + `adel` (стек)
3. `aliya` + `elena` (стек)
4. `guzel-course` + (одиночно, либо парой с ближайшим коротким)
5. `evdokia` (одиночный)
6. `china-truck` (одиночный)
7. `first-parcel` (одиночный)
8. `kate` (одиночный)

Точное распределение в коде задаётся массивом `mobileSlides: Testimonial[][]` — каждый внутренний массив = один слайд (1 или 2 карточки в столбец с `gap-4`).

### Что меняем в `src/components/landing/Testimonials.tsx`

- Добавить рядом с `testimonials` вспомогательный массив `mobileSlides` — массив групп (по 1–2 элемента), сгруппированных вручную для оптимальной компоновки.
- В мобильной карусели рендерить `mobileSlides.map(...)` вместо `testimonials.map(...)`. Внутри `CarouselItem` — flex-колонка, каждый элемент группы оборачивается в ту же `<figure>` с золотой кавычкой и рамкой, что и сейчас.
- Слайд с двумя короткими отзывами держит ту же ширину `basis-[85%]`, высота естественная — пустого пространства снизу больше нет.
- Десктопная masonry-сетка **не меняется** (как и просил пользователь — «в большой версии ничего не меняй»).
- Стрелки и точки-индикаторы продолжают работать: `count` теперь = количеству слайдов (`mobileSlides.length`), а не отзывов.

### Технические детали

- Только один файл: `src/components/landing/Testimonials.tsx`.
- Без новых зависимостей, без новых ассетов.
- Доступность: каждая `<figure>` сохраняет свой `alt` на `<img>`, ключи карусели — индекс группы.
